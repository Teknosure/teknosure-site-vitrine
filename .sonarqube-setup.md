# 🔍 Configuration SonarQube pour Teknosure Website

## 📋 Prérequis

- Docker et Docker Compose installés sur votre VPS
- Au moins 2 GB de RAM disponible pour SonarQube
- Ports 9000 disponible sur votre VPS

## 🚀 Installation sur VPS

### 1. Augmenter la limite de mémoire virtuelle (Important!)

Sur votre VPS, exécutez :

```bash
# Temporaire (jusqu'au redémarrage)
sudo sysctl -w vm.max_map_count=262144

# Permanent (survit au redémarrage)
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
```

### 2. Démarrer SonarQube

```bash
# Démarrer uniquement SonarQube et sa base de données
docker-compose up -d sonarqube sonarqube-db

# Vérifier les logs
docker-compose logs -f sonarqube
```

### 3. Accéder à SonarQube

- Ouvrez votre navigateur : `http://votre-vps-ip:9000`
- **Login par défaut** : `admin` / `admin`
- ⚠️ **Changez le mot de passe immédiatement !**

### 4. Créer un Token d'analyse

1. Connectez-vous à SonarQube
2. Allez dans **My Account** (icône utilisateur en haut à droite)
3. **Security** > **Generate Token**
4. Nom du token : `teknosure-website`
5. Type : `Project Analysis Token`
6. **Copiez et sauvegardez le token généré !**

## 🔧 Configuration locale pour l'analyse

### 1. Installer SonarScanner

**Sur macOS :**
```bash
brew install sonar-scanner
```

**Sur Linux :**
```bash
# Télécharger SonarScanner
wget https://binaries.sonarsource.com/Distribution/sonar-scanner-cli/sonar-scanner-cli-5.0.1.3006-linux.zip
unzip sonar-scanner-cli-5.0.1.3006-linux.zip
sudo mv sonar-scanner-5.0.1.3006-linux /opt/sonar-scanner
sudo ln -s /opt/sonar-scanner/bin/sonar-scanner /usr/local/bin/sonar-scanner
```

### 2. Créer un fichier de configuration local

Créez un fichier `.env.sonar` (à NE PAS commiter) :

```env
SONAR_HOST_URL=http://votre-vps-ip:9000
SONAR_TOKEN=votre-token-genere
```

### 3. Lancer l'analyse

```bash
# Avec les variables d'environnement
sonar-scanner \
  -Dsonar.host.url=http://votre-vps-ip:9000 \
  -Dsonar.login=votre-token-genere

# Ou en utilisant npm
npm run sonar -- \
  -Dsonar.host.url=http://votre-vps-ip:9000 \
  -Dsonar.login=votre-token-genere
```

## 🔒 Sécuriser SonarQube (Production)

### Option 1 : Ajouter SonarQube derrière Nginx

Créez un fichier `nginx/sonarqube.conf` :

```nginx
server {
    listen 80;
    server_name sonar.votredomaine.com;

    location / {
        proxy_pass http://sonarqube:9000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Option 2 : Restriction par IP

Modifiez `docker-compose.yml` pour ne pas exposer le port publiquement :

```yaml
sonarqube:
  ports:
    - "127.0.0.1:9000:9000"  # Accessible uniquement en local
```

Puis utilisez un tunnel SSH pour y accéder :
```bash
ssh -L 9000:localhost:9000 user@votre-vps
```

## 📊 Interpréter les résultats

SonarQube vous donnera :

- **🔴 Bugs** : Erreurs potentielles dans le code
- **🔒 Vulnerabilities** : Failles de sécurité (XSS, injection, etc.)
- **⚠️ Code Smells** : Mauvaises pratiques
- **📈 Coverage** : Couverture de tests (si configuré)
- **🔄 Duplications** : Code dupliqué

### Qualité Gates

- **A** : Excellent
- **B** : Bon
- **C** : Moyen
- **D** : Faible
- **E** : Mauvais

## 🔄 Intégration CI/CD

Pour automatiser l'analyse à chaque commit, vous pouvez ajouter dans votre pipeline :

```yaml
# Exemple pour GitHub Actions
- name: SonarQube Scan
  run: |
    npm install -g sonarqube-scanner
    sonar-scanner \
      -Dsonar.host.url=${{ secrets.SONAR_HOST_URL }} \
      -Dsonar.login=${{ secrets.SONAR_TOKEN }}
```
